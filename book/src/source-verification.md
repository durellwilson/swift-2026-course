# Source Verification

## Authoritative Reference Standards

### Primary Sources

**Apple Official Documentation**
- **URL**: developer.apple.com
- **Verification**: Direct from Apple servers
- **Update Frequency**: Real-time with releases
- **Reliability**: 100% authoritative
- **Usage**: All API references, guidelines

**WWDC Sessions**
- **URL**: developer.apple.com/videos
- **Verification**: Official Apple content
- **Update Frequency**: Annual + special events
- **Reliability**: 100% authoritative
- **Usage**: Feature announcements, best practices

**Swift Evolution**
- **URL**: github.com/apple/swift-evolution
- **Verification**: Official Swift language proposals
- **Update Frequency**: Continuous
- **Reliability**: 100% authoritative
- **Usage**: Language feature documentation

### Academic Sources

**IEEE Xplore**
- **Access**: Institutional or paid subscription
- **Peer Review**: Double-blind review process
- **Citation Format**: IEEE standard
- **Usage**: Performance research, algorithms

**ACM Digital Library**
- **Access**: Institutional or paid subscription
- **Peer Review**: Rigorous academic standards
- **Citation Format**: ACM standard
- **Usage**: Mobile computing, HCI research

**arXiv Computer Science**
- **Access**: Open access
- **Peer Review**: Pre-print with community review
- **Citation Format**: arXiv identifier
- **Usage**: Cutting-edge research, ML papers

### Industry Sources

**Engineering Blogs**
- **Verified Publishers**: Google, Meta, Netflix, Airbnb, Uber
- **Validation**: Cross-reference with production data
- **Citation**: Author, company, publication date
- **Usage**: Real-world case studies, scale patterns

**Open Source Projects**
- **Repositories**: GitHub verified organizations
- **Validation**: Code review, star count, maintenance
- **Citation**: Repository URL, commit hash
- **Usage**: Implementation examples, patterns

**Community Forums**
- **Apple Developer Forums**: Official support
- **Swift Forums**: Language discussions
- **Stack Overflow**: Verified answers only
- **Usage**: Problem-solving, edge cases

### Verification Checklist

**Before Citing Any Source**
- [ ] Author credentials verified
- [ ] Publication date within 2 years (or timeless)
- [ ] Cross-referenced with 2+ sources
- [ ] Tested in current Xcode version
- [ ] No conflicts with Apple guidelines
- [ ] Peer review status confirmed
- [ ] Accessibility considerations included

### Source Hierarchy

**Priority Order**
1. Apple official documentation
2. WWDC sessions and sample code
3. Swift Evolution proposals
4. Peer-reviewed academic papers
5. Verified industry engineering blogs
6. Open-source production code
7. Community forums (verified answers)

### Red Flags

**Reject Sources That**
- Lack author attribution
- Use deprecated APIs without warning
- Make unverified performance claims
- Contradict Apple guidelines
- Promote insecure practices
- Ignore accessibility
- Are >3 years old without validation

### Citation Format

**Apple Documentation**
```
Apple Inc. (2025). "SwiftUI Documentation."
Apple Developer Documentation.
https://developer.apple.com/documentation/swiftui
Accessed: December 2025.
```

**WWDC Session**
```
Apple Inc. (2025). "What's New in Swift."
WWDC 2025, Session 101.
https://developer.apple.com/videos/play/wwdc2025/101
```

**Academic Paper**
```
Smith, J., & Jones, M. (2024). "Performance Optimization
in Mobile Applications." IEEE Transactions on Mobile Computing,
23(4), 567-589. DOI: 10.1109/TMC.2024.1234567
```

**Engineering Blog**
```
Johnson, A. (2025). "Scaling iOS Apps to Millions of Users."
Netflix Tech Blog. Published: March 15, 2025.
https://netflixtechblog.com/scaling-ios-apps
```

### Update Protocol

**When Sources Change**
1. Monitor Apple release notes
2. Check Swift Evolution status
3. Validate code examples
4. Update citations
5. Add deprecation warnings
6. Provide migration paths
7. Document in changelog

### Archival

All sources archived with:
- Original URL
- Snapshot date
- PDF backup
- Key excerpts
- Verification notes
- Related sources

### Transparency

Source verification logs include:
- Verification date
- Verifier identity
- Validation method
- Test results
- Confidence level
- Review schedule
